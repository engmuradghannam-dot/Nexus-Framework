## متبقٍ من تقرير المراجعة (لم يُوزَّع بعد)

- تدوير/إلغاء توكن GitHub المكشوف سابقاً في المحادثات — إجراء يدوي على GitHub، خارج نطاق الكود.
- تنظيف تاريخ Git من كلمة المرور القديمة `ghannam2020` (تصحيح الكود الحالي تم، لكنها لا تزال بتاريخ commits سابقة). عملية حساسة (rewrite history + force-push) — تحتاج موافقة صريحة قبل التنفيذ.
- اختبارات لوحدات: التصنيع، المبيعات (selling)، المشتريات (buying/purchasing)، الموارد البشرية، CRM.
- ملاحظة معمارية: `Invoice` (invoicing) لا يملك حقل `company` — دالة `post_to_ledger` تستخدم أول شركة بترتيب الـ id بدل شركة الفاتورة الفعلية؛ يعمل بشكل صحيح مع شركة واحدة (كما هو الوضع الحالي) لكنه سيُصبح خطأ منطقي حقيقي بمجرد وجود أكثر من شركة/مستأجر ببيانات فعلية. يحتاج إضافة الحقل + migration، خارج نطاق تعديل بسيط.
- مراجعة متخصصة لصحة منطق المحاسبة/التصنيع تحت الحمل الفعلي.
